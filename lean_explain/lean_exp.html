Lean and Machine-Checked Proofs

Proof assistants, mathlib, and AI help for formalization

A short guide to starting with Lean

History of proof assistance I

Goal: make proofs checkable by machine. Definitions, theorem statements, and inferences must be clear enough for a small kernel to verify.

Early systems

  • Automath began in the 1960s and used typed languages to represent proofs.
  • Mizar started around 1973 and kept formal mathematics close to ordinary writing.
  • The lesson remains: a proof is structured data that a program can check.

Source: Automath.

History of proof assistance II

Major proof assistants

  • Coq/Rocq, Isabelle, HOL, ACL2, Agda, and Lean support formal proof.
  • Coq grew from work on the Calculus of Constructions starting in 1984. It is now called the Rocq Prover.
  • Lean launched in 2013. Lean 4 works well with mathlib.

Sources: Rocq Prover, Lean reference.

What changes in formal proof?

Definitions

Every object has a precise type. Hidden paper conventions become named definitions, instances, or assumptions.

Theorems

A theorem statement is a type. A proof is a term of that type. If it type-checks, Lean accepts it.

Inference

A tactic can supply each inference. The kernel checks the resulting proof object.

Lean

  • Lean launched in 2013.
  • Lean 4 is both a theorem prover and a functional programming language.
  • The official VS Code extension gives interactive support.
  • Lean is widely used for formalizing mathematics.

Mathlib

  • Mathlib is Lean's large community library for formalized mathematics.
  • It covers algebra, topology, analysis, probability, category theory, combinatorics, and more.
  • Reuse matters: each theorem can help future projects.

Sources: Lean language reference, mathlib overview.

Installation: tools

  • Install VS Code.
  • Install the Lean 4 VS Code extension.
  • Install Git.
  • Create or use a GitHub account.
  • Enable GitHub Copilot in VS Code and sign in with GitHub.

Practical note

The Lean extension shows goals, errors, docs, and tactic feedback. This is the main working interface.

Sources: Lean install guide, VS Code Copilot setup.

Create a Lean + mathlib project

mkdir lean-projects
cd lean-projects

lake +leanprover-community/mathlib4:lean-toolchain new MyMathlibProject math
cd MyMathlibProject

lake exe cache get
lake build
code .
  • math creates a Lean project with mathlib.
  • lake exe cache get downloads precompiled mathlib files to save time.
  • lake build checks the project and its dependencies.

Source: Using mathlib4 as a dependency.

Workflow: ask AI for Lean code

Give AI

  • The theorem statement.
  • Needed imports.
  • The proof outline or blueprint.
  • Nearby mathlib and project lemmas.
  • Expected assumptions and variable types.

Ask for

  • Lean code without sorry.
  • Helper lemmas, if useful.
  • Short names matching the blueprint.
  • No new axioms unless requested.

AI can help with search and repair. Lean remains the checker, so generated proof code must compile.

Workflow: compile and repair

lake build
  • If the build succeeds, Lean accepted the proof.
  • If the build fails, send AI the error and goal state.
  • Repeat: generate, build, inspect, repair.

Useful error data

Include the exact file location, local context, target goal, and failed tactic message. This is usually better than a prose summary.

Check axioms and assumptions

#print axioms theorem_name
  • If sorryAx appears, a sorry placeholder remains.
  • Check for unwanted custom axioms.
  • Check that assumptions match the paper statement.
  • Use lake build. For stronger validation, use lean4checker when needed.
Rule of thumb

A compiled file is not enough if the theorem statement was weakened. Compare the Lean assumptions with the intended claim.

Source: Validating a Lean Proof.

Why is this important?

Proof checking

Human refereeing can take months. Once formalized, Lean checks the proof automatically.

Gap finding

Formalization reveals gaps, missing assumptions, unclear definitions, and notation conflicts.

Reuse

Once in mathlib, a theorem can be imported and reused.

A growing library also helps AI theorem proving. Models and search tools get more verified examples and reusable lemmas.

Journal submission?

Ordinary math journals

Lean files are usually not required. For example, Annals of Mathematics asks for manuscript files such as a PDF and arXiv-style source.

Formalization journals

Formalization journals expect code artifacts. Annals of Formalized Mathematics asks for formal proof artifacts and code.

Sources: Annals of Mathematics submission guidelines, Annals of Formalized Mathematics instructions.
